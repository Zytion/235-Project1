Process

Day 1

Bobby and Mira discussed the concept for the game and refined the concepts of Bobby's initial proposal. Mira made the base outline for the classes, creating Resources, Population, and Upgrades. Bobby worked on creating an image that represented how everything would be laid out, then Bobby and Mira worked together to finalize the design. Bobby started working on the DOM layout of all the sections of the game.

Day 2

Mira started initializing the game loop in main.js, creating the variables that would be used for the game. She used a ticker to allow for constant updates regularly while Bobby continued to work on the DOM positioning.

Day 3

Mira worked on creating the methods that the game would use to hook up to buttons in the game. She also worked on starting the localStorage saving for all the variables neccessary. Bobby finished the layout of the DOM and all the elements (buttons, hover over text).

Day 4

Bobby refined the JavaScript code and hooked up most of the buttons and values so they would appear in the DOM. Mira worked on new JavaScript classes for tool upgrades versus population upgrades, she worked on more methods that the game would use as well.

Day 5

Both Mira and Bobby work to finish the rest of the methods, hooking them up to the DOM, for the review day.

Day 6

Taking the advice from the review day, Bobby adds more instructions to the game with more text that pops up when you hover over game objects. Mira starts thinking about extra concepts they could add to the game, deciding upon events that could be both good or bad.

Day 7

Mira adds the event methods while Bobby continued to work on the base game, refining the JavaScript code further and fixing the game blanace. He also calculated and then fixed the amount of resources used for upgrades and buildings.

Day 8

Bobby finished up the game, ensuring code is validated and removing any of the remaining bugs. Mira double-checked the rubric for requirements and confirmed that the game follows all the requirements.
